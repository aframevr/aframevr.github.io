Primitives

Introduction

Primitives are concise, semantic building blocks blocks that wrap A-Frame’s underlying entity-component system. A-Frame ships with a handful built-in primitives for common use cases such as <a-cube>, <a-model>, and <a-sky>. These primitives are to help people get started with using A-Frame. To uncover the full composability and extensibility of A-Frame, dive down into the underlying entity-component system.

Primitives are designed to enable developers to start building A-Frame scenes without having to learn A-Frame’s full entity-component system. By being semantic and concise, they also improve the readability of our markup.

Primitives and entity-component instances can be freely mixed within an A-Frame scene. Entities with components are useful when developers need to go beyond primitives and tap into the deeper power and flexibility of A-Frame’s built-in components, which include material, geometry, light, and more.
